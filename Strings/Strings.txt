/// Mutable and Immutable string

// Immutable string: String
// - Once created, cannot be modified
// - Any modification creates a new object
// - Example: String s = "Hello";

// Mutable string: StringBuffer, StringBuilder
// - Can be modified (append, insert, delete, etc.) without creating new objects

// StringBuilder : 
// - Mutable sequence of characters
// - Not synchronized (not thread-safe)
// - Faster, suitable for single-threaded applications

// StringBuffer : 
// - Mutable sequence of characters
// - Synchronized (thread-safe, safe to use in multi-threaded apps)
// - Slightly slower than StringBuilder due to synchronization


// ðŸ”¹ 1. String
// Immutable â†’ Once created, it cannot be changed.
// Any modification (like concat, replace, etc.) creates a new object in memory.
// Thread-safety: Not relevant, since immutability already makes it safe to use across threads.
// Use case: Constant text (e.g., "Hello World")

// //ðŸ”¹ 2. StringBuffer
// Mutable â†’ You can modify the content (append, insert, delete).
// Thread-safe because all its methods are synchronized.
// Slower than StringBuilder due to synchronization overhead.
// Use case: When text needs to change often in a multi-threaded environment.

// //ðŸ”¹ 3. StringBuilder
// Mutable â†’ You can modify the content (append, insert, delete).
// Not thread-safe, so it's faster than StringBuffer in single-threaded scenarios.
// Use case: When text needs to change often in a single-threaded environment.
